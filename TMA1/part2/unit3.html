Basic Fundamentals of XML and AJAX

XML is used for data transfer and data storage.

The term Ajax was coined by Jesse James Garrett of Adaptive Path in February 2005.

AJAX is used for asynchronous communication between the server and the client-side without reloading the whole page. It uses a combination of XMLHttpRequest object to request from the server and the server responds back with a JSON object.

XML:

  • XML stands for eXtensive Markup Language. It is an open-source technology for data storage and exchange. It describes data in a way that humans can easily read and understand them and computers can process.

  • XML uses a Document Type Definition, which defines the structure and validates the elements and attributes of an XML document.

  • The correct syntax to define the XML version is: <?xml version="1.0"?>.

  • Every XML element must be properly closed. There will be errors when you don't close an element. White spaces are also preserved with XML.

  • XSL stands for eXtensive Stylesheet Language. It formats and transforms XML documents.

AJAX:

  • AJAX stands for Asynchronous JavaScript and XML. It is a way to send and receive information from the server to the pages without reloading the whole page.

  • AJAX can create an XMLHttpRequest object to request data from the server, then the server sends back a JSON object to the client-side.

  • Raw Ajax uses JavaScript to send asynchronous requests to the server, then updates the page using DOM.

  • The main steps of using AJAX technology are: Create an instance of the XMLHttpRequest object, then handle the server's response as JSON object. Parse this JSON object using the syntax JSON.parse() and now you can work with the data received.
